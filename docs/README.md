# Documentation Reading Order

The recommended order for reading the documentation files.

## Recommended Reading Sequence

1. **booting.md**
   - Start here to understand the overall boot process of the Raspberry Pi and how the firmware loads and starts the kernel.

2. **cross_compiler.md**
   - Learn how to set up the cross-compiler toolchain required to build the kernel and related binaries for ARM64.

3. **exception_level.md**
   - Read this to understand the ARM exception levels (EL2, EL1, etc.) and how the CPU transitions between them during boot.

4. **exceptions.md**
   - This file explains the different types of exceptions, how they are handled, and the role of the exception vector table.

5. **memory_management.md**
   - Finish with this file to get a detailed understanding of memory management, virtual memory, MMU configuration, and page tables.
